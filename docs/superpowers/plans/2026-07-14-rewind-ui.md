# Rewind UI Implementation Plan

> **For agentic workers:** Implement task-by-task. Spec: `docs/superpowers/specs/2026-07-14-rewind-ui-design.md`

**Goal:** Empty-input double Esc → Rewind list → confirm with Restore / Summarize from / Summarize up to / Never mind.

**Architecture:** Pure helpers in `rewind-panel.ts` + `session-history` turn indices; core `summarizeTranscriptRange`; overlay mode in `terminal-layout`; actions wired from `chat.ts`.

### Task 1: Core summarize range

- Modes: `from_here` (keep `[0, endExclusive)`, summarize `[endExclusive, length)` → one summary msg) and `up_to_here` (summarize `[0, start)`, keep `[start, length)`).
- Summary message: `role: "assistant"`, `metadata: { rewindSummary: true }`, content = formatted L1 draft (+ optional context note).
- Also update L1 via `consolidateToL1` on the collapsed slice when helpful; always rewrite L0.

### Task 2: Turn index helper

- From transcript, list display user turns with `{ text, timestamp, transcriptIndex }` where `transcriptIndex` is the L0 index of that user message.
- Append synthetic `(current)` only in the UI layer (not from helper).

### Task 3: Rewind panel + Esc overlay + chat wiring

- Double Esc empty → open list; Enter current → close; Enter history → confirm; actions call core + reload UI.

### Task 4: Tests

- Core range tests; panel render/nav; Esc arm coexistence with clear.
